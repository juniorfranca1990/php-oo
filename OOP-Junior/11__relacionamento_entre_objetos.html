Relacionamento entre objetos

Em OO é comum que objetos se relacionem um com os outros para desempenhar suas tarefas. Os relacionamentos existentes: ASSOCIAÇÃO | AGRAGAÇÃO | COMPOSIÇÃO

Associação: É a relação mais comum entre dois objetos. Quando acontece? Quando uma classe “usa” outra classe. Exemplo: Um motorista utiliza (depende do carro, mas este não faz parte do motorista) o carro para trabalhar.

Agregação: É o tipo de relação conhecida como todo/parte. Na agregação o objeto agrega outro objeto, ou seja, torna um objeto externo parte de si.. Assim o objeto pai poderá utilizar funcionalidades do objeto agregado, mas ele não depende dele (agregado) para existir. Exemplo: A tampa de uma caneta é parte da caneta, se a caneta não tiver a tampa, ela não deixará de ser uma caneta.

Composição: Neste caso, o objeto-pai é responsável pela criação e destruição de suas partes. O objeto-pai realmente “possui” as instâncias de suas partes. Resumidamente, um só existe caso o outro também exista.

Exemplo: O motor de um carro é parte fundamental do carro.

ASSOCIAÇÃO 

	class Produto{

		public $nome;
		public $fornecedor;

		public function __construct($nome, $fornecedor){
			$this->nome = $nome;
			$this->fornecedor = $fornecedor;
		}
	}
	class Fornecedor{

		public $razaoSocial;
		public $endereco;

		public function __construct($razaoSocial, $endereco){
			$this->razaoSocial = $razaoSocial;
			$this->endereco = $endereco;
		}
	}
	$fornecedor = new Fornecedor("Mercado pague-menos", "Rua C");

	$produto = new Produto("Arroz", $fornecedor);

	echo "Produto: ".$produto->nome;
	echo "Fornecedor: ".$produto->fornecedor->razaoSocial;
